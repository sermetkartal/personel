# OCR Service — Phase 2 Benchmark Methodology

## Objectives

Phase 2 benchmarks must validate two targets defined in the architecture:

1. Inference latency: p95 < 500 ms per screenshot (1920x1080, typical office UI).
2. Throughput: >= 10 screenshots/second on a 4-vCPU server (sustained).
3. Accuracy (WER): Word Error Rate <= 5% on the synthetic Turkish business corpus.

## Benchmark Corpus

Synthetic screenshots are generated rather than captured from real employee
workstations to avoid KVKK implications during benchmarking.

### Generation approach

```
bench/generate_corpus.py  (Phase 2 — not yet implemented)
```

Script renders PNG images using:
- `pillow` + `reportlab` to render Turkish business text at various font sizes.
- A vocabulary drawn from `bench/vocab/tr_business.txt` (200 common phrases).
- Three resolution tiers: 720p / 1080p / 1440p.
- Three noise levels: clean / medium JPEG artefact / low brightness.
- 100 images per tier/noise combination = 900 total benchmark images.

### PII exclusion

The synthetic corpus NEVER contains real TCKNs, IBANs, or credit card numbers.
Test strings that deliberately trigger redaction are only used in
`tests/test_redaction.py` as synthesised, unit-level inputs — never in images.

## Benchmark Harness

```bash
# Phase 2 — scaffold only; implementation in bench/run.py

python bench/run.py \
    --url http://localhost:8080/v1/extract \
    --corpus bench/corpus/ \
    --concurrency 4 \
    --output bench/results/$(date +%Y%m%d).json
```

Harness records:
- Wall-clock latency per request (p50 / p95 / p99 / max)
- Throughput (RPS)
- Engine reported confidence (mean / min)
- Word count per image (to detect gross extraction failures)

## Accuracy Measurement (WER)

Word Error Rate is computed by comparing OCR output against ground-truth text
embedded in each synthetic image (generated by the same script that draws the
text, so it is exact).

```
WER = (S + D + I) / N
```

Where S = substitutions, D = deletions, I = insertions, N = words in reference.

Target: WER <= 5% on the clean tier; <= 15% on the low-brightness tier.

## Hardware Reference

All Phase 2 benchmarks run on:
- 4 vCPU (AMD EPYC or Intel Xeon), 8 GB RAM
- Tesseract 5.x, `tur+eng` LSTM engine, OEM 3 PSM 3
- Docker container resource limits: 2 CPU / 4 GB RAM (matching compose config)
